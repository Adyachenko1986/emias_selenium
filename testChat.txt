1. Во время авторизации системы автоматически осуществляется переход в чат (в параметрах урла указан Ticket)
Вы на своей стороне: 1) вызываете метод проверки ticket 2) вызываете метод получения информации по врачу по уникальному идентификатору пользователя (из ticket)

2. Переход из МКАБ (медицинской карты)
Формируем url для перехода как определились с параметрами: 1) Ticket врача, по пациенту: 2) ФИО, 3) полис, 4) дата рождения 5) СНИЛС, 6) ссылка на ЭМК пациента в региональном сервисе ИЭМК.
В результате перехода автоматически создаются кейсы(чаты) с данными (с сообщениями).

3. Переход из ТАП
Формируем url для перехода как определились: 1) Ticket врача, по пациенту: 2) ФИО, 3) полис, 4) дата рождения 5) СНИЛС, 6) ссылка на ЭМК пациента в региональном сервисе ИЭМК 7) номер ТАП
В результате перехода автоматически создаются кейсы(чаты) с данными (с сообщениями).
